8:3 PRIORITY ENCODER

Eight input lines compress into a 3-bit binary code. Multiple inputs can be active at once — the highest-indexed line always wins, exactly like a real interrupt controller (e.g. the 74148 IC).

Interactive Logic Schematic

Why the expressions look complicated: unlike a plain encoder, a priority encoder must guarantee only the highest active line counts — each term for A1 and A0 includes NOT-conditions on every higher-priority line, which is exactly what the "P" priority-winner indicator in the diagram is doing for you visually.
